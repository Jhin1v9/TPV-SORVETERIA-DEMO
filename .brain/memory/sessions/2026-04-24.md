# 202604241630 - Bug Detector kiosk restored to native flow
#bug-detector #kiosk #product #observability #devops

**Contexto:** O kiosk estava com um provider customizado que abria um modal manual e escondia recursos nativos do pacote `@auris/bug-detector`.

**Problema:** A experiencia perdeu o fluxo de inspecao primeiro e depois report, alem de cortar capacidades importantes como screenshot anotavel, video, session replay e painel nativo de acompanhamento.

**Decisao:** Voltar para a arquitetura do adapter original em `headless`, mantendo o motor oficial do pacote e adicionando apenas uma camada leve de comandos rapidos no shared.

**Implementado:**
- `BugDetectorOverlay` nativo para hover + clique em elemento
- `BugReportModal` nativo com screenshot, anotacao e video
- `BugTrackerPanel` nativo com replay, previews e status
- comandos rapidos para `Inspecionar`, `Relatorios` e `Print`
- `printAllReports()` via export HTML do pacote
- workspace lateral com owner, status, prioridade e comentarios
- integracao Kimi em modo compativel com OpenAI usando `https://api.moonshot.ai/v1`
- enriquecimento automatico por IA com titulo, resumo, passos de reproducao, impacto e direcao de fix

**Configuracao Kimi:**
- usar `VITE_MOONSHOT_API_KEY` ou `VITE_KIMI_API_KEY`
- `VITE_KIMI_MODEL` padrao: `kimi-k2.5`
- `VITE_MOONSHOT_BASE_URL` padrao: `https://api.moonshot.ai/v1`
- decisao baseada na documentacao oficial que informa compatibilidade com formato OpenAI e uso do modelo `kimi-k2.5`

**Racional de produto:** Seguindo `personas/product.md`, o detector precisa resolver a jornada completa de captura, contexto e handoff, nao apenas abrir um formulario.

**Racional de observabilidade:** Seguindo `personas/devops.md`, o detector passa a ser uma camada de observabilidade operacional no front, com export e artefatos mais ricos para reproduzir erros.

**Proximo passo recomendado:** mover a chamada Kimi para um backend/proxy seguro para nao expor chave no client e adicionar sincronizacao real de comentarios/status entre usuarios.

# 202604241200 - Restauracao RPCs + Fixes Criticos
#supabase #rpc #bug-fix #surgeon #devops #e2e

**Contexto:** Projeto Supabase anterior (`jmvikjujftidgcezmlfc`) com cache PostgREST permanentemente corrompido. Todas as RPCs retornavam 404 mesmo existindo no Postgres.

**Persona ativa:** The Surgeon + The DevOps

## Diagnostico
- Script `FIX_RPCS.sql` (497 linhas) falhava silenciosamente no SQL Editor do Supabase
- SQL Editor nao processa scripts grandes com multiplos blocos `$$`
- Funcoes pequenas (5 linhas) funcionavam; scripts grandes diziam "Success" mas nao criavam nada
- `NOTIFY pgrst, 'reload schema'` nao resolveu
- Restart do projeto no dashboard nao resolveu

## Decisao: Novo projeto Supabase
- Criado projeto `dproxlygtabihfhtxdvm`
- Aplicado schema-expanded.sql (tabelas + reset_demo_data)
- RPCs divididas em 9 arquivos SQL separados em `supabase/migrations/`

## Bug PostgREST Cache — Descoberta Critica
- Funcoes `upsert_customer`, `upsert_store_settings`, `reset_demo_data` ficavam INVISIVEIS na API REST (404)
- Mesmo apos DROP + CREATE + restart do projeto
- **Causa raiz:** PostgREST cache bug conhecido — certos nomes de funcoes ficam permanentemente travados
- **Solucao:** Renomear funcoes:
  - `upsert_customer` → `save_customer`
  - `upsert_store_settings` → `save_store_settings`
  - `reset_demo_data` → `restore_demo_data`

## Codigo alterado
- `.env.local`: nova URL e chave do projeto
- `packages/shared/src/realtime/client.ts`: 3 nomes de RPC atualizados
- `supabase/migrations/`: 9 arquivos SQL para recriacao manual

## BUG-005 — Kiosk carousel preto entre transicoes (FIXED)
**Root cause:** `AnimatePresence mode="wait"` fazia a imagem anterior sair COMPLETAMENTE antes da nova entrar, revelando o fundo escuro do gradiente overlay por ~1.2s.
**Fix:** Substituído `AnimatePresence mode="wait"` por renderizacao estatica de todas as imagens com animacao de `opacity` e `scale` cruzada via `motion.div`. Cada slide mantem seu `z-index` (ativo=1, inativo=0). Transicao `duration: 1.4s` com `easeInOut`.
- Nunca mais ha momento sem imagem visivel (overlap garantido)
- `loading={idx === 0 ? 'eager' : 'lazy'}` para performance
